# 상점봇 README

## 실행

```bash
# 멘션 처리 봇
ruby main.rb

# 툿 카운트 정산 스케줄러 (별도 프로세스)
ruby toot_scheduler.rb
```

PM2 사용 시:
```bash
pm2 start main.rb --name shop_bot --interpreter ruby
pm2 start toot_scheduler.rb --name toot_scheduler --interpreter ruby
```

---

## 명령어

| 명령어 | 설명 |
|--------|------|
| `[등록/캐릭터명]` | 신규 등록. 스탯 포인트 10pt 지급 |
| `[구매/아이템명]` | 크레딧으로 아이템 구매 |
| `[스탯구매/스탯명]` | 스탯 포인트 1pt 소모해 스탯 1 증가 |
| `[소지품]` | 크레딧 / 스탯 포인트 / 스탯 / 아이템 확인 |
| `[사용/아이템명]` | 아이템 사용 (인벤토리에서 제거) |
| `[양도/아이템명/@상대ID]` | 아이템 양도 |
| `[송금/금액/@상대ID]` | 크레딧 송금 |

스탯명 입력 시 별칭도 가능:
- 건강 / 체력
- 마법능력 / 공격
- 인내 / 방어
- 속도 / 민첩
- 기술 / 명중
- 행운 / 크리티컬

## 크레딧 획득

| 방법 | 지급량 |
|------|--------|
| 100툿당 자동 정산 (매일 자정) | 15크레딧 |
| 필수 진행 참여 (운영자 수동 지급) | 50크레딧 |
| 출석 (운영자 수동 지급) | 5크레딧 |

## 스프레드시트 구조

### 사용자 시트
| 열 | 항목 |
|----|------|
| A | ID (마스토돈 acct) |
| B | 이름 |
| C | 크레딧 |
| D | 아이템 (콤마 구분) |
| E | 메모 |
| F | 마지막베팅일 |
| G | 오늘베팅횟수 |
| H | 마지막타로일 |
| I | 누적툿수 (마스토돈 statuses_count) |
| J | 정산기준툿수 (마지막 정산 시점) |
| K | 스탯포인트잔여 |

### 스탯 시트
| 열 | 항목 | 기본값 |
|----|------|--------|
| A | ID | - |
| B | 이름 | - |
| C | 건강 (체력) | 50 |
| D | 마법능력 (공격) | 10 |
| E | 인내 (방어) | 10 |
| F | 속도 (민첩) | 0 |
| G | 기술 (명중) | 0 |
| H | 행운 (크리티컬) | 5 |

### 아이템 시트
| 열 | 항목 |
|----|------|
| A | 이름 |
| B | 설명 |
| C | 가격 (크레딧) |
| D | 판매여부 (TRUE/FALSE) |
| E | 사용가능여부 (TRUE/FALSE) |
